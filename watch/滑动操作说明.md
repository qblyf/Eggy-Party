# 📱 滑动操作说明

## 🎮 双页面设计

手表版现在采用左右滑动的双页面设计，更符合手表的操作习惯！

### 页面1：抽取页面（主页）

```
┌─────────────────┐
│  🥚 蛋仔盲盒    │
├─────────────────┤
│ [统计信息]      │
│                 │
│    [盲盒]       │
│                 │
│  [抽取按钮]     │
│                 │
│  [结果卡片]     │
│                 │
│ ← 滑动查看收藏→ │
└─────────────────┘
```

**功能：**
- 查看统计数据
- 点击盲盒抽取
- 查看抽取结果
- 向左滑动查看收藏

### 页面2：收藏页面

```
┌─────────────────┐
│  📦 我的收藏    │
├─────────────────┤
│ [统计卡片]      │
│                 │
│ [筛选标签]      │
│                 │
│ [卡] [卡] [卡]  │
│ [卡] [卡] [卡]  │
│ [卡] [卡] [卡]  │
│                 │
│ ← 滑动返回抽取→ │
└─────────────────┘
```

**功能：**
- 查看收藏统计
- 筛选不同稀有度
- 查看所有蛋仔
- 向右滑动返回抽取

## 🎯 操作方法

### 在手表上

1. **左右滑动切换页面**
   - 在屏幕上向左滑 → 进入收藏页面
   - 在屏幕上向右滑 → 返回抽取页面

2. **点击页面指示器**
   - 屏幕底部有两个小圆点
   - 点击圆点可以快速切换页面

3. **抽取蛋仔**
   - 在抽取页面点击盲盒
   - 或点击"抽取盲盒"按钮

4. **查看收藏**
   - 滑动到收藏页面
   - 点击筛选标签查看不同稀有度
   - 点击卡片查看详情（如果实现）

### 在电脑上测试

1. **鼠标拖拽**
   - 按住鼠标左键
   - 向左或向右拖动
   - 松开鼠标切换页面

2. **点击指示器**
   - 点击底部的圆点切换页面

## ✨ 设计特点

### 符合手表操作习惯

- ✅ **左右滑动** - 手表最自然的操作方式
- ✅ **单页面显示** - 避免滚动，内容一目了然
- ✅ **大按钮** - 易于点击
- ✅ **清晰分页** - 功能明确，不混乱

### 流畅的动画

- ✅ **平滑过渡** - 使用 cubic-bezier 缓动函数
- ✅ **跟手滑动** - 实时跟随手指移动
- ✅ **智能回弹** - 滑动不足自动回弹
- ✅ **阈值判断** - 滑动超过30%才切换页面

### 视觉反馈

- ✅ **页面指示器** - 显示当前在哪一页
- ✅ **滑动提示** - 提示用户可以滑动
- ✅ **不同背景** - 两个页面有不同的渐变背景

## 🎨 页面背景

### 抽取页面
- 渐变色：金黄 → 粉红 → 浅粉
- 温暖活泼的色调

### 收藏页面
- 渐变色：紫蓝 → 深紫
- 沉稳优雅的色调

## 📊 页面指示器

位置：屏幕底部中央

```
● ○  ← 在抽取页面
○ ●  ← 在收藏页面
```

- 白色实心圆 = 当前页面
- 半透明圆 = 其他页面
- 可点击切换

## 💡 使用技巧

### 快速切换

1. **快速滑动**
   - 快速向左/右滑动
   - 即使滑动距离短也能切换

2. **点击指示器**
   - 最快的切换方式
   - 直接点击想去的页面

### 查看收藏

1. **抽到新蛋仔后**
   - 向左滑动查看收藏
   - 确认是否已收集

2. **筛选查看**
   - 点击筛选标签
   - 只看特定稀有度

### 连续抽取

1. **停留在抽取页面**
   - 不需要切换页面
   - 连续点击抽取

2. **查看结果**
   - 结果直接显示在抽取页面
   - 不影响操作流程

## 🔧 技术实现

### 触摸事件

- `touchstart` - 记录起始位置
- `touchmove` - 实时更新位置
- `touchend` - 判断是否切换页面

### 鼠标事件（电脑测试）

- `mousedown` - 记录起始位置
- `mousemove` - 实时更新位置
- `mouseup` - 判断是否切换页面

### 滑动阈值

- 滑动距离 > 30% 屏幕宽度 → 切换页面
- 滑动距离 < 30% 屏幕宽度 → 回弹

### 性能优化

- 使用 `transform` 而非 `left/right`
- 启用硬件加速 `will-change: transform`
- 平滑的缓动函数
